A Diagnostic Evaluation of Small Language Models for Offline Socratic Tutoring in Brazilian Portuguese: A Study on Structural and Pedagogical Adherence under Public-School Infrastructure Constraints

8 open SLMs × 13 canonical scenarios × 3 repetitions = 312 calls — plus a 76-call falsification protocol — all strictly offline on CPU.

1. The central finding, in plain terms

This study asked a precise question: can off-the-shelf Small Language Models (≤ 3.8 B parameters) operate as Socratic tutors of writing in Brazilian Portuguese, under the same infrastructure constraints faced by a typical Brazilian public school?

The answer breaks into two parts. First, only 5 out of 8 models obey the structured-output contract specified in natural language under zero-shot regime — the others either reformat the output as enumerated lists or substitute the schema entirely. Second, even the structurally conformant model (qwen2.5:3b-instruct) mobilizes the linguistic terminology required by the rubric in only 46 % of scenarios, drifting toward thematic-narrative commentary on the literary work cited rather than examining the student's own writing.

The unexpected nuance: when probed with a targeted four-isolation falsification protocol, the failure of the Llama 3.2 family — 78/78 divergent zero-shot — collapses entirely under one in-context demonstration: the same family reaches 100 % conformance with a single example pair (Fisher exact: p < 10⁻¹¹). The disobedience is real but reversible; the family is capable, just biased.

2. How the experiment was constructed

The pipeline is deliberately spare — no GPU, no cloud, no grammar coercion. The goal is to measure what off-the-shelf SLMs natively do when handed a pedagogical contract in plain Portuguese, under conditions a Brazilian public school could afford.

Five design choices that shape every number that follows

1
Local and offline. Every call goes to a local Ollama server at localhost:11434, with models quantized to Q4_K_M. No network, no proprietary API. This emulates the computational ceiling of a typical public-school computer lab in Brazil.
2
Single canonical SYSTEM_PROMPT. All 312 calls receive identical instructions, covering role (Socratic tutor for 8th–9th grade), pedagogical constraints (no ready answers, value linguistic varieties), and output schema (JSON with pontos_fortes as string and perguntas_reflexivas as a list of strings).
3
Thirteen canonical scenarios. Drafted by a specialist in Koch's Textual Linguistics. Each scenario embeds one cohesive phenomenon to be diagnosed: referential repetition, ambiguous pronominalization, contradictory connectives, temporal-marker disorganization, juxtaposition without logical anchoring.
4
Three repetitions per pair. temperature = 0.2. The replication enables a coefficient-of-variation estimate, which distinguishes expected sampling noise from systemic instruction instability.
5
Deterministic four-condition validator. The function divergente() classifies each response by JSON syntax, root type, pontos_fortes typology, and perguntas_reflexivas cardinality+typology. No LLM judge — the property being measured is syntactic-typological and admits no interpretive subjectivity.

The experimental pipeline

The validator algorithm, in actual Python

def divergente(raw_response: str) -> bool:
    try:
        parsed = json.loads(raw_response)
    except json.JSONDecodeError:
        return True                                      # C1: invalid JSON

    if not isinstance(parsed, dict):
        return True                                      # C2: root not object

    pf = parsed.get("pontos_fortes")
    if not (isinstance(pf, str) and pf.strip()):
        return True                                      # C3: wrong type or empty

    pr = parsed.get("perguntas_reflexivas")
    if not isinstance(pr, list) or len(pr) == 0 \
       or any(not isinstance(x, str) for x in pr):
        return True                                      # C4: wrong type, empty, or non-string

    return False

This function is the gatekeeper. Every "divergent" cell in every chart below comes from this exact code, applied to the raw model output preserved in the data files.

How to read this table

The three structurally divergent models (100 % column) make the JSON output unusable for downstream parsing — the contract specified in the SYSTEM_PROMPT is violated in every single call. The two ceiling models (Llama 3 8B, Gemma 2 9B) confirm that obedience is achievable at scale, but neither fits the 8 GB RAM ceiling of a public-school machine. qwen2.5:3b-instruct is the only deployable model that combines structural obedience with latency below 10 s.

phi3:mini is the most pathological outlier on three axes simultaneously: 100 % divergence, p95 latency of 43.7 seconds, and verbosity nearly four times higher than peers (471 tokens vs. ~120). Microsoft's marketing of the model as "strong in structured reasoning" did not survive a Brazilian Portuguese socratic-tutoring prompt.

4. The falsification protocol — why Llama 3.2 isn't actually "broken"

A 78/78 failure rate is too uniform to accept at face value. Before publishing the architectural claim, four competing hypotheses were tested in isolation, totaling 80 additional calls under the same Ollama stack. Each isolation falsifies one alternative explanation.

E1 — Stochasticity isolated

Repeat at temperature = 0. If the failure were sampling noise, it would change. Result: 0 / 26 conformant. Identical to baseline. Hypothesis falsified.

E2 — Compositional incapacity isolated

Prepend a single user/assistant demonstration of the canonical schema. If the model lacked the capacity, it would still fail. Result: 23 / 23 conformant. Hypothesis falsified.

E3 — Python client layer isolated

Send the identical JSON payload via curl, bypassing the Python requests library entirely. Result: failure reproduced verbatim. Hypothesis falsified.

E2b — Reversal stability isolated

Combine one-shot with temperature = 0. If the E2 reversal were noise, it would collapse. Result: 21 / 24 conformant (3 timeouts excluded). Hypothesis falsified.

What the four isolations jointly establish

The Llama 3.2 family does have the compositional capacity to obey the schema. What is missing is obedience under zero-shot instruction in natural language — an attractor of the post-training process. The architecture is fine; the alignment isn't. Adding a single demonstration in context overrides the attractor and restores conformance to 100 %.

5. Inferential statistics — Fisher's exact, seven contingencies

Each categorical claim of the paper is backed by a 2×2 Fisher exact test (two-sided). The test is chosen over χ² because the data exhibits cells with zero count — the asymptotic χ² approximation is invalid under such degeneracy. Fisher's exact computes the probability by combinatorial enumeration of more extreme outcomes; no distributional assumption is required.

How to interpret these p-values

The first five contrasts (F1–F5) reach probabilities below 10⁻²¹ — these are not "borderline significant", they are among the most categorical statistical separations observable in 2×2 experimentation. The two paired-condition reversals (F6, F7) at 10⁻¹¹ and 10⁻¹² confirm that the recovery under one-shot is overwhelmingly unlikely under the null hypothesis of no effect.

Note that the odds ratio is formally degenerate (one cell is zero) for all seven contrasts. The reportable effect is the p-value alone, in line with standard practice for categorical comparisons under extreme proportion regimes.

6. Reproduce in four commands

Requires Python 3.11+, Ollama installed locally, ~8 GB of RAM and ~30 GB of disk for the eight model weights.

# 1) Pull all eight models locally
ollama pull qwen2.5:1.5b-instruct qwen2.5:3b-instruct \
            llama3.2:1b llama3.2:3b \
            gemma2:2b gemma2:9b \
            llama3:8b phi3:mini

# 2) Install Python dependencies
pip install -r requirements.txt

# 3) Run the main benchmark (8 × 13 × 3 = 312 calls)
python src/benchmark_local.py

# 4) Reproduce the inferential analysis (Fisher exact on 7 contingencies)
python src/inferential_statistics.py

For the falsification protocol on the Llama 3.2 family (4 isolations, 80 calls):

python src/counter_experiment_llama32.py

All raw data, scripts, the paper in three formats, and this interactive report are versioned at github.com/RandMelville/slm-socratic-tutor-ptbr.
